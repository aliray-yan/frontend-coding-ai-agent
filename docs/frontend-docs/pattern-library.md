# Frontend Pattern Library

Navbar:

- Desktop: logo, primary nav, utility links, primary action.
- Mobile: compact top bar, menu button, full-width disclosure, clear active states.

Hero:

- Use a clear headline, concise supporting copy, and one primary CTA.
- Keep media relevant to the product, portfolio, or page goal.
- Make the next section partially visible when the hero is first viewport content.

Cards:

- Use stable dimensions and predictable padding.
- Keep titles concise and body copy scannable.
- Avoid nesting cards inside cards.

Portfolio:

- Lead with role or specialty, proof of work, selected projects, process, testimonials, and contact.
- Project cards should show stack, outcome, and a direct action.

E-commerce:

- Product page: gallery, title, price, rating/reviews, variants, quantity, add to cart, delivery/returns, product details, recommendations.
- Collection page: filters, sort, product grid, active filter chips, pagination or infinite scroll.
- Cart page: item rows, quantity controls, totals, discounts, checkout CTA, trust messaging.

Dashboard:

- Prioritize dense, calm scanning.
- Use tables, filters, tabs, segmented controls, and restrained metrics.
- Avoid landing-page hero composition inside operational tools.
