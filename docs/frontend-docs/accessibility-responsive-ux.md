# Accessibility, Responsive Design, and UX Review Notes

Accessibility:

- Use semantic landmarks: header, nav, main, section, footer.
- Keep heading order logical.
- Use buttons for actions and links for navigation.
- Add visible focus states.
- Keep contrast strong for text, controls, and badges.
- Provide labels for inputs and accessible names for icon buttons.

Responsive design:

- Test mobile, tablet, laptop, and wide desktop.
- Keep touch targets at least 44px where possible.
- Avoid text overlap by using flexible containers, wrapping, min-width control, and sensible max widths.
- Use `aspect-ratio` for fixed-format media and cards.
- Check that sticky headers, sidebars, and modals do not trap content on small screens.

UI/UX review:

- Visual hierarchy should make the primary action and main content obvious.
- Spacing should use a consistent rhythm.
- Typography should match the density of the surface.
- Color should communicate roles: primary action, secondary action, warning, danger, success, disabled.
- Layout balance should support scanning and repeated use.
